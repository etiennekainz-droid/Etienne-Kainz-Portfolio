# Etienne Kainz — Engineering Portfolio

Dark, document-style portfolio ("technical dossier" aesthetic) for mechanical &
aerospace engineering work. **Vanilla HTML / CSS / JS** — no build step — deploys
to GitHub Pages by pushing the files.

```
site/
├── index.html            # 01 — Home (hero, floating lattice anim, tensile-test dashboard, contents)
├── projects.html         # 02 — Projects (2.1 … 2.6, verbatim descriptions, annex PDFs)
├── drawings.html         # 03 — Technical drawings & concepts
├── aerial.html           # 04 — Aerial photography
├── certifications.html   # 05 — Certifications (incl. CU Boulder quantum specialization)
├── styles.css            # design system (IBM Plex, hairline rules, title block)
├── shared.js             # nav, reveals, lightbox
├── anim.js               # canvas figures: 3-D lattice wave (interactive) + tensile-test dashboard
├── .nojekyll             # keep — stops GitHub Pages from running Jekyll
└── assets/               # images & PDFs (regenerated by ../build-assets.sh)
```

All paths are **relative**, so the site works at `username.github.io` or
`username.github.io/repo-name` without configuration.

## Run locally
```bash
cd site
python3 -m http.server 8080     # → http://localhost:8080
```

## Deploy to GitHub Pages (link for LinkedIn)
```bash
cd site
git init && git add . && git commit -m "Portfolio"
git branch -M main
git remote add origin https://github.com/<you>/<repo>.git
git push -u origin main
```
GitHub → **Settings → Pages** → Source: *Deploy from a branch* → `main` / `/ (root)`.
The public URL (`https://<you>.github.io/<repo>/`) is what you put on LinkedIn —
it opens directly in the browser like any website.

> Optional, for a nicer LinkedIn link preview: after deploying, add absolute
> `og:url` / `og:image` meta tags in `index.html` using your final URL.

## Editing content
- Text lives directly in the HTML pages (each page is a numbered "section").
- To add an image to a project gallery: drop the file in `assets/projects/<id>/`
  and add a `<figure>` (visible) or `<span hidden data-lb-group=…>` (lightbox-only)
  in `projects.html`.
- `../build-assets.sh` rebuilds `assets/` from the source folders — one folder per
  project/section, folders are the single source of truth (wipes first, so
  deleting a source image removes it from the site).
- Animations are in `anim.js`; they pause off-screen and respect
  `prefers-reduced-motion`.
